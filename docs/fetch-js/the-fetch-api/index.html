The Fetch API

Reading ⏱ 2 min read

The Fetch API is a built-in browser feature that lets JavaScript send HTTP requests. It replaces the older XMLHttpRequest API and is now the standard way to communicate with web services.

The core function is simply fetch(url), which returns a Promise — an object representing a result that will arrive in the future.

1. Promises, async and await#

Because fetching data takes time, fetch is asynchronous — it doesn't block the rest of your code while waiting for the server. A Promise represents the eventual result.

The cleanest way to work with Promises is the async / await syntax:

  • async before a function means the function can use await inside it.
  • await before a Promise pauses that function until the Promise settles, then returns the resolved value.
Two awaits are needed

The first await waits for the response headers to arrive. The second await waits for the full response body to be read and parsed as JSON. Both steps are asynchronous.
