JSON

Reading ⏱ 2 min read

JavaScript Object Notation (JSON) is a lightweight text format used to store and exchange data between systems. It is the standard format returned by almost every web API.

JSON represents data as key-value pairs wrapped in curly braces {}. Keys are always strings, and values can be strings, numbers, booleans, arrays, objects, or null.

1. JSON Data Types#

Supported value types in JSON
TypeExampleDescription
String"hello"Text wrapped in double quotes
Number42 or 3.14Integer or decimal number
Booleantrue or falseA yes/no value
Array[1, 2, 3]An ordered list of values
Object{"x": 1}A nested set of key-value pairs
NullnullRepresents an empty or missing value

2. Arrays in JSON#

A JSON array is an ordered list of values enclosed in square brackets []. Each item is separated by a comma. Arrays are commonly used when an API returns multiple records — for example, a list of products or a set of search results.

3. Nested Objects#

Objects can contain other objects, allowing you to represent complex, hierarchical data. A value for one key can itself be a {} object with its own key-value pairs.

JSON vs JavaScript Objects

JSON looks like a JavaScript object, but there are key differences: JSON keys must always be in double quotes, and JSON cannot contain functions or comments.
