Handling Errors

Reading ⏱ 2 min read

Network requests can fail. The server might be down, the URL might be wrong, or the response might indicate an error. Writing code that handles these failures gracefully is essential for building reliable applications.

1. try / catch#

The try / catch statement lets you run code that might throw an error and respond to failures without crashing the program.

  • Code inside the try block runs normally.
  • If an error is thrown, execution jumps to the catch block, where you can log the problem or show a message to the user.

2. Checking response.ok#

As noted in the HTTP & Responses section, fetch only throws for network failures. A 404 or 500 response is still a successful Promise resolution — you must check response.ok yourself and decide what to do.

Best practice

Always check response.ok inside a try/catch when writing production code. This handles both network failures and HTTP error codes in one place.
