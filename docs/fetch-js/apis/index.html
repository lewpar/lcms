APIs

Reading ⏱ 2 min read

An Application Programming Interface (API) is a defined way for two pieces of software to communicate. One program makes a request, and the other responds with data or an action.

On the web, APIs are used everywhere. When you check the weather, load a map, or log in with Google, your browser is talking to an API behind the scenes.

1. Web APIs#

A web API (also called a REST API) communicates over the internet using URLs. You request a specific URL called an endpoint, and the server responds with data — almost always in JSON format.

For example, the public DummyJSON API exposes this endpoint:

https://dummyjson.com/users/1

Requesting that URL returns a JSON object containing information about user number 1.

2. HTTP Methods#

Every API request uses an HTTP method that describes what kind of action is being performed. The most common methods are:

Common HTTP methods
MethodPurposeExample
GETRead / retrieve dataLoad a user's profile
POSTSend / create new dataSubmit a registration form
PUTReplace existing dataUpdate a user's full profile
PATCHPartially update dataChange just a user's email
DELETERemove dataDelete a post
In this module we focus on GET

The fetch() function defaults to a GET request, which is used to read data. You will learn how to send other methods later in your studies.

3. Endpoints and Parameters#

An endpoint is a specific URL that the API exposes. Different endpoints return different data. Many APIs also accept query parameters — extra values appended to the URL after a ? — to filter or customise the response.
