Event Listeners & Interactivity

DOM & Events ⏱ 2 min read

1. What are Events?#

An event is something the user does on a web page, like:

  • Clicking a button
  • Typing in a text field
  • Hovering over an element
  • Pressing a key
  • Scrolling the page

With event listeners, you can detect when these things happen and run code in response.

Common Events:

  • click - User clicks an element
  • input - User types in a text field
  • keypress - User presses a key
  • mouseover - Mouse moves over element
  • mouseout - Mouse leaves element
  • submit - Form is submitted

2. Adding Event Listeners#

Use the addEventListener() method to respond to events:

let button = document.querySelector("button");

button.addEventListener("click", function() {
  console.log("Button was clicked!");
});

Or with arrow functions (modern way):

let button = document.querySelector("button");

button.addEventListener("click", () => {
  console.log("Button was clicked!");
});

Syntax: element.addEventListener(eventType, function)

  • eventType - The event to listen for (as a string, no quotes around function)
  • function - The code to run when the event happens

3. The Event Object#


When an event happens, JavaScript passes an event object to your function. This object contains useful information:


let button = document.querySelector("button");

button.addEventListener("click", (event) => {
  console.log(event.type);     // "click"
  console.log(event.target);   // The button element
});

let input = document.querySelector("input");

input.addEventListener("input", (event) => {
  console.log(event.target.value);  // What the user typed
});

let form = document.querySelector("form");

form.addEventListener("submit", (event) => {
  event.preventDefault();  // Stop form from submitting
  console.log("Form prevented from submitting!");
});



Useful properties:


    

  • event.target - The element that triggered the event
    • 

  • event.type - The type of event (click, input, etc.)
    • 

  • event.target.value - The current value (for inputs)
    • 

  • event.preventDefault() - Stops default behavior

4. Combining Events and DOM Changes#


The real power comes when you respond to events by changing the DOM:


let button = document.querySelector("#toggleButton");
let message = document.querySelector("#message");

button.addEventListener("click", () => {
  message.textContent = "You clicked the button!";
  message.style.color = "green";
});



You can create interactive features:


    

  • Toggle elements on/off
    • 

  • Update text based on user input
    • 

  • Change colors/styles on interaction
    • 

  • Add/remove elements dynamically
