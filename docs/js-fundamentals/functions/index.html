1.6. Functions#

A function is a reusable block of code that performs a specific task. Instead of writing the same code multiple times, you write it once in a function and call it whenever you need it.

1.6. Declaring Functions#

Use the function keyword to declare a function. After the function name, write the code block inside curly braces.

1.6. Parameters and Arguments#

Functions can accept parameters (variables in the function definition) to customize their behavior. When you call the function, you pass arguments (actual values).

1.6. Return Values#

Use return to send a value back from the function. Once return is executed, the function stops.

1.6. Arrow Functions#

A shorter way to write functions using the => syntax. Useful for simple functions.

1.6. Default Parameters#

You can set default values for parameters. If the caller doesn't provide a value, the default is used.

1.6. Why Use Functions?#

  • Reusability: Write once, use multiple times
  • Organization: Group related code together
  • Easier debugging: Fix a bug in one place
  • Readability: Give code a meaningful name
  • Modularity: Break complex tasks into smaller pieces
